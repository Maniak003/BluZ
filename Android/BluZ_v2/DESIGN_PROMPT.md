# Промпт для Claude Design — BluZ UI Redesign

Скопируй и вставь в Claude Design (claude.com).

---

```
Мне нужно переработать интерфейс приложения BluZ — портативный 
сцинтилляционный гамма-спектрометр для Android. Приложение должно 
поддерживать обе ориентации: ландшафтную и портретную, адаптируясь 
к каждой.

ТЕКУЩАЯ СТРУКТУРА (6 вкладок):
1. Спектр — живой график гамма-спектра (1024/2048/4096 каналов), 
   масштабирование щипком, переключение лог/линейная шкала, 
   кнопка старт/стоп, кнопка сохранения
2. История — накопленный спектр за время измерения, те же элементы 
   управления графиком
3. Дозиметр — гистограмма 512 каналов, показывает распределение дозы
4. Лог — журнал событий прибора (временные метки + коды событий, 
   прокручиваемый список)
5. Настройки — конфигурация прибора: пороги уровней (level1/2/3), 
   переключатели звука/LED/вибро, высокое напряжение, компаратор, 
   калибровочные коэффициенты (9 чисел), таймер спектрометра
6. Карта — GPS-карта (Yandex MapKit) с цветными метками радиационного 
   фона, запись трека, кнопка текущего положения

СТАТУСНАЯ СТРОКА (всегда видна):
- Индикатор BT-соединения (красный/жёлтый/зелёный)
- CPS (импульсов в секунду)
- Мощность дозы (мкР/ч или мкЗв/ч)
- Температура детектора (°C)
- Напряжение батареи (В)
- Время измерения

ПРОБЛЕМЫ КОТОРЫЕ НУЖНО РЕШИТЬ:
- Навигация между вкладками неудобна — свайп сложно обнаружить, 
  подписи вкладок слишком мелкие
- Экран настроек перегружен — слишком много элементов сразу
- Статусная строка плохо читается с первого взгляда
- Нет чёткой иерархии между основными данными (спектр, доза) 
  и второстепенными элементами управления
- Сигнал перегрузки детектора недостаточно заметен

ТРЕБОВАНИЯ К ДИЗАЙНУ:
- Поддержка обеих ориентаций — Android адаптивный layout
- В портретной ориентации: навигация снизу, статус сверху, 
  график занимает центр
- В ландшафтной ориентации: навигация сбоку (rail), статус 
  в компактной строке сверху, график занимает максимум площади
- Тёмная тема (используется в полевых условиях при слабом освещении)
- Крупные элементы управления (работа в перчатках)
- Нижняя/боковая панель навигации вместо вкладок-свайпов
- Настройки разбить на разделы: Оповещения, Оборудование, Калибровка
- Статусная строка: иконки + значения, компактно
- Состояние перегрузки: яркое красное предупреждение

ОСОБЕННОСТИ ЭКРАНА:
- Поддержка вырезов (notch/punch-hole) под камеру — контент не должен 
  перекрываться вырезом ни в одной ориентации
- В ландшафте вырез обычно смещается влево или вправо — статусная 
  строка и навигация должны это учитывать
- Скруглённые углы экрана — важные элементы (кнопки, индикаторы) 
  не должны попадать в зону скругления
- Использовать WindowInsets API: statusBars, navigationBars, 
  displayCutout — все отступы от системных элементов должны 
  применяться через insets, не хардкодом

Спроектируй:
1. Главный экран (спектр) в портретной ориентации
2. Главный экран (спектр) в ландшафтной ориентации
3. Экран настроек
4. Состояние перегрузки
```
