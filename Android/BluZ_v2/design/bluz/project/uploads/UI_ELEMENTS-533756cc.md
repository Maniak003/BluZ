# BluZ — Описание всех элементов интерфейса

Документ описывает каждый экран, каждый элемент управления и каждое отображаемое значение.  
Для каждого элемента указано: **что показывает / что делает**, **может ли пользователь это менять**.

---

## Общая структура приложения

Приложение состоит из двух зон:

1. **Основная область** (слева, занимает большую часть экрана) — текущий активный экран из 6 вкладок
2. **Правая боковая панель** (узкая вертикальная полоса) — навигация + статус, видна всегда

---

## Правая боковая панель (постоянная)

### Индикатор BT-соединения

| Поле | Описание |
|------|----------|
| Тонкая цветная полоска вверху панели | Состояние Bluetooth-подключения к прибору |
| Красный | Не подключено |
| Жёлтый | Идёт подключение |
| Зелёный | Подключено, данные идут |

**Пользователь не меняет** — только читает.

---

### Статусные строки (текст над навигацией)

Видны на вкладках: Спектр, История, Дозиметр. На остальных вкладках скрыты.

#### Строка 1 — температура, напряжение, время
```
25 C  3.75v  Time:00:02:15:00
```
| Значение | Источник | Изменяемость |
|----------|----------|--------------|
| Температура SiPM-детектора, °C | Прибор → BLE-пакет | Только чтение |
| Напряжение батареи, В | Прибор → BLE-пакет | Только чтение |
| Время измерения, чч:мм:сс | Прибор → BLE-пакет | Только чтение |

#### Строка 2 — суммарный счёт
```
Total:1234567(85.23%)  AvgCPS:423.50
```
| Значение | Источник | Изменяемость |
|----------|----------|--------------|
| Суммарное число импульсов | Прибор | Только чтение |
| Процент заполнения буфера спектрометра | Расчёт | Только чтение |
| Среднее CPS (скользящее среднее) | Прибор | Только чтение |

#### Строка 3 — текущий CPS и мощность дозы
```
CPS:156 (10.34 uR/h)  Avg:9.87 uR/h
```
или, при превышении детектора:
```
Overload  ← ярко-розовый/пурпурный цвет
```
| Значение | Источник | Изменяемость |
|----------|----------|--------------|
| CPS — импульсов в секунду | Прибор | Только чтение |
| Мощность дозы (uR/h или uSv/h) | CPS × коэффициент из настроек | Только чтение |
| Avg — среднее значение | Расчёт | Только чтение |
| **Overload** (пурпурный) | Прибор, бит 3 байта 97 | Только чтение — критическое состояние |

**Единицы измерения** (uR/h или uSv/h) выбираются в настройках.

#### Строка COMPared/MED — компаратор/медиана
Видна только на вкладках Спектр и История.  
Показывает дополнительную статистику по графику. **Только чтение.**

#### Строка изотопа
Видна только на вкладках Спектр и История.  
Показывает информацию об определённом изотопе (по положению курсора). **Только чтение.**

## Экран 1 — Спектр

Показывает живой гамма-спектр, обновляется каждую секунду от прибора.

### Отображаемые элементы

| Элемент | Что показывает | Изменяемость |
|---------|----------------|--------------|
| График (весь экран) | Текущий гамма-спектр: ось X — каналы (0–1024/2048/4096), ось Y — число импульсов в канале | Только чтение (но масштабируется жестами) |
| Курсор (линия) | Вертикальная линия в точке касания, показывает номер канала и энергию | Управляется касанием |

### Жесты на графике

| Жест | Действие |
|------|----------|
| Щипок (pinch) | Масштабирование по оси X |
| Тап | Установка курсора на канал |
| Свайп | Горизонтальная прокрутка при зуме > 1 |

### Кнопки управления (правая часть экрана)

| Кнопка | Надпись | Что делает | Изменяет прибор? |
|--------|---------|------------|-----------------|
| `buttonSpecterSS` | **Start** / **Stop** | Запускает или останавливает накопление спектра в приборе | Да — команда приборy |
| `buttonSaveBQ` | **BqMon** | Сохраняет текущий спектр в файл формата BqMon (.bqm) | Нет |
| `buttonLoadSpectr` | **Load** | Загружает ранее сохранённый спектр из файла для просмотра | Нет (данные в приборе не меняются) |
| `buttonClearSpectr` | **Clear** | Очищает накопленный спектр (сброс в приборе) | Да — команда приборy |
| `buttonCalibrate` | **1** | Включает режим калибровки: следующий клик на пике задаёт опорную энергию | Нет (до подтверждения) |
| `buttonConfirmCalibrate` | **X** | Подтверждает или отменяет калибровку | Да — обновляет коэффициенты |

### Чекбоксы обработки

| Элемент | Надпись | Что делает |
|---------|---------|------------|
| `cbSMA` | **S** | Включает сглаживание SMA (Simple Moving Average) на графике |
| `cbMED` | **M** | Включает медианный фильтр на графике |

Оба чекбокса **меняет пользователь**, влияют только на отображение (прибор не затрагивается).

> Есть ещё `cbMLEM` (MLEM-алгоритм) и прогресс-бар — в настоящее время скрыты (`visibility=invisible`), не используются.

## Экран 4 — Лог

Два прокручиваемых журнала рядом.

### Левый журнал: события прибора (`logsText`)

Хранит до 50 последних событий. Показывает дату/время и тип события.  
**Только чтение.** Цветовая кодировка:

| Цвет | Событие |
|------|---------|
| Белый | Turn on — включение |
| Жёлтый | Level 1 — превышение порога 1 |
| Красный | Level 2 — превышение порога 2 |
| Пурпурный | Level 3 — превышение порога 3 |
| Зелёный | Normal — возврат к норме |
| Белый | Dosimeter reset, Spectrometer reset, Write to flash |
| Белый | Start/Stop spectrometer, Clear logs |
| Белый | Change resolution, Change bits of chan |
| Ярко-розовый | **Overload** — перегрузка детектора |
| Ярко-розовый | **Low battery** — низкий заряд |
| Белый | Calibrate Batt, Clear history |
| Ярко-розовый | **Need upgrade** — требуется обновление прошивки |

### Правый журнал: лог приложения (`appLogText`)

Отладочный лог самого приложения. Цветовая кодировка по уровню:

| Цвет | Уровень |
|------|---------|
| Красный | Ошибка (0) |
| Синий | Информация (1) |
| Жёлтый | Предупреждение (2) |
| Зелёный | Успех (3) |
| Тёмно-серый | Отладка (4) |
| Пурпурный | Отладка 2 (5) |

**Уровень видимости** задаётся в Настройках → Log level (0 = ничего не показывать).

### Кнопки

| Кнопка | Надпись | Что делает | Изменяет прибор? |
|--------|---------|------------|-----------------|
| `buttonClearLog` | **Clear** | Очищает журнал событий прибора (на экране + команда приборy) | Да |
| `buttonClearAppLog` | **AppClr** | Очищает лог приложения (только экран) | Нет |
| `buttonSaveAppLog` | **Save** | Сохраняет лог приложения в файл | Нет |

## Экран 5 — Настройки

Разбит на две колонки, обе прокручиваются.

---

### Левая колонка: «Application parameters» (параметры приложения)

Все настройки сохраняются в SharedPreferences. Применяются к отображению, файлам, интерфейсу.

#### Подраздел: Подключение

| Элемент | Тип | Описание | Изменяет прибор? |
|---------|-----|----------|-----------------|
| `textMACADDR` | EditText | MAC-адрес Bluetooth прибора. Формат: `XX:XX:XX:XX:XX:XX`. Используется для фильтрации при BLE-сканировании. | Нет (только настройка поиска) |

#### Подраздел: Specter histogram (стиль отображения спектра)

| Элемент | Тип | Описание |
|---------|-----|----------|
| `rgTypeSpectr` | RadioGroup | Тип графика: **Line** (линия) или **Gisto** (гистограмма) |
| Шкала Y | RadioGroup (4 кнопки) | **Lin** — линейная, **Lg** — логарифмическая, **FoneLin** — линейная с фоном, **FoneLg** — лог с фоном |
| `tvColor` | ImageView | Превью выбранного цвета линии спектра |
| `seekBarA` | SeekBar 0–255 | Прозрачность (Alpha) линии спектра |
| `seekBarR` | SeekBar 0–255 | Красный канал цвета линии |
| `seekBarG` | SeekBar 0–255 | Зелёный канал |
| `seekBarB` | SeekBar 0–255 | Синий канал |
| `editTextSMAWindow` | EditText (число) | Окно SMA-сглаживания (кол-во точек, default=5). Чем больше — тем сильнее сглаживание. |
| `editTextRejectCann` | EditText (число) | Отбрасывать первые N каналов от нуля (default=10). Убирает шум на низких каналах. |

Все — **пользователь меняет**, на прибор не влияют.

#### Подраздел: Export format (формат сохранения спектра)

| Элемент | Тип | Описание |
|---------|-----|----------|
| `rbBqMon` | RadioButton | Сохранять спектр в формате BqMon (.bqm) |
| `rbSPE` | RadioButton | Сохранять спектр в формате CSV |

#### Подраздел: Interface

| Элемент | Тип | Описание |
|---------|-----|----------|
| `CBFullScreen` | CheckBox | Полноэкранный режим (скрывает системную строку) |
| `CBNightMode` | CheckBox | Ночной режим карты (тёмная тема Yandex MapKit) |
| `editTextPaddingLeft` | EditText (число) | Отступ слева для области рисования спектра (пикселей) |
| `editTextPaddingRight` | EditText (число) | Отступ справа |
| `editTextXZoom` | EditText (decimal) | Горизонтальный зум графика (default=1.0, можно 0.5–4.0) |

#### Подраздел: Map format (формат экспорта треков)

| Элемент | Тип | Описание |
|---------|-----|----------|
| `rbKML` | RadioButton | Экспортировать трек в KML (Google Earth) |
| `rbGPX` | RadioButton | Экспортировать трек в CSV |

#### Подраздел: Application logs

| Элемент | Тип | Описание |
|---------|-----|----------|
| `editTextApplucationLog` | EditText (число) | Уровень логирования: 0=отключено, 1=информация, 2=предупреждения, 3=успех, 4=отладка, 5=детальная отладка |

#### Подраздел: Units of measurement

| Элемент | Тип | Описание |
|---------|-----|----------|
| `rbURH` | RadioButton | Показывать мощность дозы в мкР/ч (uR/h) |
| `rbUSVH` | RadioButton | Показывать мощность дозы в мкЗв/ч (uSv/h). 1 uSv/h ≈ 100 uR/h |

#### Подраздел: Energy compensation (калибровка детектора)

| Элемент | Тип | Описание |
|---------|-----|----------|
| `textDetectorName` | TextView | Имя активного детектора (только чтение) |
| `buttonNewDetector` | Button | Создать новый профиль детектора в базе данных |
| `buttonLoadDetector` | Button | Загрузить параметры детектора из файла |
| `buttonSelectDetector` | Button | Выбрать активный профиль детектора из списка |

### Правая колонка: «Device parameters» (параметры прибора)

Настройки, которые читаются из прибора кнопкой **Read** и записываются в прибор кнопкой **Write**.

#### Клик на импульс (Click)

| Элемент | Описание |
|---------|----------|
| `rbClickNone` — none | Звук при импульсе отключён |
| `rbClick1` — 1/1 | Звуковой сигнал на каждый импульс |
| `rbClick10` — 1/10 | Звуковой сигнал на каждый 10-й импульс |

#### LED на импульс

| Элемент | Описание |
|---------|----------|
| `rbLedNone` — none | LED при импульсе отключён |
| `rbLed1` — 1/1 | LED мигает на каждый импульс |
| `rbLed10` — 1/10 | LED мигает на каждый 10-й |

#### Авто-старт

| Элемент | Тип | Описание |
|---------|-----|----------|
| `CBSpectrometer` | CheckBox | Автоматически запускать спектрометр при включении прибора |

#### Подраздел: Calibrate Ax² + Bx + C (калибровочные коэффициенты)

Полином пересчёта номера канала в энергию (кэВ). Три набора — для каждого разрешения (1024, 2048, 4096).

| Элемент | Тип | Описание |
|---------|-----|----------|
| `editPolA` | EditText (decimal, signed) | Коэффициент A (при x²) |
| `editPolB` | EditText (decimal, signed) | Коэффициент B (при x) |
| `editPolC` | EditText (decimal, signed) | Свободный член C |

> Применяется к текущему разрешению (выбранному в Res ниже).

#### Подраздел: Alarms (пороговые уровни тревоги)

| Элемент | Тип | Описание |
|---------|-----|----------|
| `editLevel1` | EditText (число) | Порог 1 в uR/h — превышение = событие Level 1 (жёлтое) |
| `CBSoudLevel1` | CheckBox | Звуковое оповещение при Level 1 |
| `CBVibroLevel1` | CheckBox | Виброоповещение при Level 1 |
| `editLevel2` | EditText (число) | Порог 2 в uR/h — Level 2 (красное) |
| `CBSoudLevel2` | CheckBox | Звук при Level 2 |
| `CBVibroLevel2` | CheckBox | Вибро при Level 2 |
| `editLevel3` | EditText (число) | Порог 3 в uR/h — Level 3 (пурпурное) |
| `CBSoudLevel3` | CheckBox | Звук при Level 3 |
| `CBVibroLevel3` | CheckBox | Вибро при Level 3 |

#### Подраздел: Hardware (низкоуровневые параметры)

| Элемент | Тип | Диапазон | Описание |
|---------|-----|----------|----------|
| `RGResolution` | RadioGroup | 1024 / 2048 / 4096 | Разрешение спектрометра (каналов). Больше — точнее, но медленнее передача |
| `editAquracy` | EditText (число) | default=100 | Точность АЦП |
| `editCPS2RH` | EditText (decimal) | — | Коэффициент пересчёта CPS → uR/h. Зависит от типа детектора |
| `editHvoltage` | EditText (число) | 0–1023 | Высокое напряжение для SiPM-детектора (10 бит) |
| `editComparator` | EditText (число) | 0–1023 | Порог компаратора (10 бит) |
| `editBitsChannel` | EditText (число) | 16–32 | Бит на канал при логарифмическом сжатии (default=20) |
| `editSampleTime` | EditText (число) | 0–7 | Время выборки АЦП (биты 0–2 байта 97 протокола) |

#### Кнопки действий (правый столбик)

| Кнопка | Надпись | Что делает | Изменяет прибор? |
|--------|---------|------------|-----------------|
| `buttonSaveSetup` | **Save** | Сохраняет все настройки в локальное хранилище (SharedPreferences) | Нет |
| `buttonRestoreSetup` | **Rest** | Восстанавливает настройки из локального хранилища в поля формы | Нет |
| `buttonScanBT` | **Scan** | Запускает BLE-поиск устройства по имени/MAC | Нет |
| `buttonReadFromDevice` | **Read** | Читает параметры ИЗ прибора → заполняет поля формы | Нет (чтение) |
| `buttonWriteToDevice` | **Write** | Записывает значения из формы В прибор (BLE-команда) | **Да** |
| `buttonFind` | **Find** | Посылает команду «ping» (cmd=6) — прибор издаёт звуковой сигнал для поиска | **Да** |

## Экран 6 — Карта

GPS-карта Yandex MapKit с цветными маркерами радиационного фона.

### Отображаемые элементы

| Элемент | Что показывает | Изменяемость |
|---------|----------------|--------------|
| Карта (весь экран) | Yandex Maps — тайловая карта местности | Жесты масштабирования/прокрутки |
| Цветные маркеры | Точки GPS с цветом по уровню радиации. 32 цвета: синий (норма) → зелёный → жёлтый → красный (превышение) | Только чтение (добавляются автоматически при записи) |
| `textCurrentTrack` | Имя текущего активного трека (зелёный текст вверху) | Только чтение |

### Цветовая шкала маркеров

32 уровня от синего (фон, clr00) до красного (высокое значение, clr31).  
Уровень выбирается пропорционально: `pulsesPerSec` / Level3 × 31.

### Кнопки

| Кнопка | Тип | Надпись | Что делает |
|--------|-----|---------|------------|
| `buttonRecord` | Button | **Record** / **Stop** | Начать/остановить запись GPS-трека с радиационными метками |
| `buttonMapLocate` | ImageButton (GPS-иконка) | — | Центрировать карту на текущем местоположении |
| `buttonAddPoint` | ImageButton (точки-иконка) | — | Добавить текущую GPS-точку вручную (без автоматической записи) |
| `buttonTracks` | Button | **Tracks** | Открыть список сохранённых треков для просмотра/выбора |
| `buttonNewTrack` | Button | **New** | Создать новый пустой трек в базе данных |
| `buttonSaveTrack` | Button | **Save** | Экспортировать активный трек в файл (KML или CSV, по настройке) |

## Аппаратные кнопки телефона

## Итоговая таблица: что может менять пользователь

### Только чтение (телеметрия от прибора)
- Температура детектора
- Напряжение батареи
- Время измерения
- Суммарные импульсы, CPS, среднее CPS
- Мощность дозы
- Состояние перегрузки (Overload)
- События в журнале прибора

### Пользователь меняет (настройки приложения, не затрагивают прибор)
- MAC-адрес для подключения
- Стиль и цвет графика спектра
- SMA-окно и отброс каналов
- Формат экспорта спектра (BqMon / CSV)
- Полноэкранный режим
- Ночной режим карты
- Отступы и зум графика
- Формат экспорта трека (KML / CSV)
- Уровень логирования
- Единицы измерения (uR/h / uSv/h)
- Управление профилями детектора

### Пользователь меняет (команды и настройки прибора, требуют Write/подключения)
- Запуск/остановка спектрометра (Start/Stop)
- Очистка спектра, истории, дозиметра, лога
- Звук на импульс (none / 1/1 / 1/10)
- LED на импульс (none / 1/1 / 1/10)
- Авто-старт спектрометра
- Калибровочные коэффициенты A, B, C (для трёх разрешений)
- Пороговые уровни Level 1/2/3 (uR/h)
- Звуковое и вибро-оповещение для каждого уровня
- Разрешение спектрометра (1024 / 2048 / 4096)
- Высокое напряжение (HV)
- Компаратор
- Биты на канал (BitsOfChan)
- Время выборки АЦП (SampleTime)
- Коэффициент CPS→uR/h

### Пользователь управляет (GPS и треки)
- Запись трека (Record / Stop)
- Центрирование карты
- Ручное добавление точки
- Просмотр, создание, сохранение треков

## Примечания для дизайна

1. **Перегрузка** — критическое состояние, требует немедленного внимания. Сейчас: только текст «Overload» пурпурного цвета в строке CPS. Нужно: яркий полноэкранный или крупный баннер.

2. **Статусная строка** видна только на 3 из 6 экранов. На Лог/Настройки/Карту она скрывается полностью.

3. **Навигация** сейчас — вертикальная боковая панель с иконками без подписей, и свайп через ViewPager2. Свайп обнаружить сложно — нет визуальных подсказок.

4. **Настройки** сейчас — одна длинная двухколоночная форма без деления на разделы с возможностью прокрутки. Очень перегружена.

5. **Кнопки Write/Read** — критически важные: Write изменяет прошивку прибора. Нужно визуально отличить от «безопасных» кнопок.

6. **Калибровочные коэффициенты** (A, B, C) — экспертные параметры. Обычный пользователь не должен их случайно менять.

7. **HV и Comparator** — аппаратные параметры низкого уровня, неправильное значение может вывести детектор из строя. Нужна защита от случайного изменения.
